1. Overview & About This App

PixKove ("we", "our", "the app") is a personal automation tool built and operated by an individual developer. It uses official APIs from social media platforms including X (Twitter), Instagram, LinkedIn, and others to automate the scheduling, posting, and performance tracking of content for a single personal brand account.

The application is built on n8n, a self-hosted workflow automation platform. Media content is sourced from Google Cloud Storage and Google Drive. The entire workflow operates under the control of a single individual for personal productivity purposes.

PixKove does not operate as a public service, SaaS product, or multi-user platform. It manages only the owner's own social media accounts.

2. Data We Collect

Because PixKove is a personal automation tool and not a user-facing product, we do not collect data from third-party users. The only data handled includes:

  • Content created and owned by PixKove (images, captions, hashtags)
  • Post performance metrics retrieved from platform APIs (impressions, likes, reach) for the owner's own posts only
  • API credentials (stored securely in the n8n instance; never logged or shared)

We do not collect, process, or store personal data belonging to any third-party users or followers.

3. How We Use Your Data

All data processed by PixKove is used exclusively for:

  • Scheduling and publishing original content to owned social media accounts
  • Retrieving analytics on published posts to improve content strategy
  • Automating routine social media management tasks for the account owner

No data is used for advertising, profiling, resale, or any commercial purpose beyond managing the owner's own brand presence.

4. Platform-Derived Data

PixKove accesses data from social media platforms strictly through their official APIs and OAuth 2.0 authorization flows. Any platform-derived data (such as post metrics) is:

  • Used solely for internal analysis by the account owner
  • Never resold, redistributed, or shared with third parties
  • Never used to build profiles of other platform users
  • Handled in compliance with each platform's Developer Agreement and Terms of Service

5. Third-Party Sharing

We do not sell, rent, trade, or share any data with third parties. The automation workflow runs entirely within privately controlled infrastructure (n8n self-hosted instance and Google Cloud services under the owner's account).

6. Data Retention

Post performance metrics may be stored temporarily in Google Sheets for analysis. No personal data belonging to other users is retained. API credentials are stored securely within the n8n instance and are never exported or logged externally.

7. Security

We take reasonable steps to protect all credentials and data:

  • All API keys and tokens stored as encrypted environment variables within n8n
  • OAuth 2.0 used for all platform authentication — no passwords stored
  • Access limited exclusively to the account owner
  • No public endpoints expose credentials or stored data
